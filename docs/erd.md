# Database Model

## Description

OrderFlow uses a relational database model focused on order management, product stock control and audit logging.

The main entity is `Order`, which represents a business request created by a user and composed of multiple products through `OrderItem`.

---

## Initial Entities

### User

Represents a system user.

Main fields:
- id
- name
- email
- password
- role
- active

### Product

Represents a product available for ordering.

Main fields:
- id
- name
- description
- sku
- stock
- price
- active

### Order

Represents a customer order.

Main fields:
- id
- customerName
- createdAt
- status
- totalAmount
- createdBy

### OrderItem

Represents a product inside an order.

Main fields:
- id
- orderId
- productId
- quantity
- unitPrice
- subtotal

### AuditLog

Represents an audit record for important actions in the system.

Main fields:
- id
- entityType
- entityId
- action
- timestamp
- userId
- details

### Notification

Represents a notification generated by the system.

Main fields:
- id
- recipient
- channel
- message
- status
- sentAt

---

## Main Relationships

- A User can create many Orders.
- An Order has many OrderItems.
- A Product can be referenced by many OrderItems.
- OrderItem acts as an intermediate entity between Order and Product.
- A User can generate many AuditLogs.
